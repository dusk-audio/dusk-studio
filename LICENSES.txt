Dusk Studio — third-party license inventory
=====================================

Dusk Studio itself is GPL-3.0-or-later (see LICENSE). This file lists every
external component bundled or vendored into the build, the license it
ships under, and the path where it lives. Update whenever a new
dependency is added so the legal surface stays auditable.

PROJECT LICENSE
---------------
Dusk Studio source code: GPL-3.0-or-later (./LICENSE)


VENDORED DEPENDENCIES (compiled into the Dusk Studio binary)
------------------------------------------------------

JUCE (audio framework)
  Version  : 8.x  (Linux pin: plugdata-team/JUCE wayland-juce8;
                   macOS/Win: upstream juce-framework/JUCE)
  License  : Dual-licensed — GPL-3.0 (used here) OR JUCE Commercial
  Path     : ../JUCE-wayland (Linux), ../JUCE (macOS/Windows)
  Upstream : https://juce.com/

Dusk Audio plugins (donor DSP cores)
  License  : GPL-3.0-or-later (matches Dusk Studio)
  Path     : ../plugins-main/plugins/  (Linux worktree)
             ../plugins/plugins/        (macOS/Windows sibling)

               - kiss_fft      BSD-3-Clause  external/sfizz/src/external/kiss_fft/
               - pugixml       MIT          external/sfizz/src/external/pugixml/
               - spline        BSD-3-Clause  external/sfizz/src/external/spline/
               - tunings       MIT          external/sfizz/src/external/tunings/
               - cpuid         BSD-3-Clause external/sfizz/src/external/cpuid/  (Steinwurf ApS)
               - hiir          WTFPL        external/sfizz/src/external/hiir/   (Laurent de Soras)

PLATFORM-SPECIFIC OPTIONAL DEPENDENCIES
---------------------------------------

LAME / libmp3lame (MP3 bounce export — optional)
  License  : LGPL-2.1-or-later
  Path     : not vendored — system shared library found at configure time via
             find_library(mp3lame); linked only when present
             (DUSKSTUDIO_HAS_LAME). Builds without it fall back to WAV-only.
  Notes    : Linked against the system SHARED library (e.g. libmp3lame.so),
             never statically — find_library resolves the distro / Homebrew
             shared object at configure time (verified: release Linux build
             links /usr/lib64/libmp3lame.so). The LGPL-2.1 §6 relink obligation
             is met by that dynamic-link boundary. Present in a binary only if
             the build was configured with libmp3lame available.
  Upstream : https://lame.sourceforge.io/


Steinberg ASIO SDK (Windows only, optional)
  License  : Steinberg ASIO SDK Licensing Agreement (proprietary)
  Path     : fetched into RUNNER_TEMP at CI time, not in tree
  Notes    : Required only for ASIO driver support on Windows.
             Distribution rights covered by Steinberg's SDK terms;
             Dusk Studio builds against the headers but does NOT redistribute
             any portion of the SDK in source or binary form.
  Upstream : https://www.steinberg.net/developers/


WHAT THIS MEANS FOR YOU
-----------------------

Self-build from source : All components compatible with GPL-3.0.
                          You can build and use Dusk Studio freely.
Paid Patreon binaries  : You are paying for support, packaging, and
                          ongoing development access. The source is
                          still GPL — your binary's compliance with the
                          GPL is unaffected by the payment.
Redistribute the binary: Allowed under GPL-3.0 terms. Include LICENSE,
                          this file, and source-code availability.
